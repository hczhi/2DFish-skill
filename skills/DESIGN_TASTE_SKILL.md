# 高能动性前端 Skill

## 1. 基线配置
* DESIGN_VARIANCE（设计变化度）: 8（1=完美对称，10=艺术混沌）
* MOTION_INTENSITY（动效强度）: 6（1=完全静态，10=电影级物理动效）
* VISUAL_DENSITY（视觉密度）: 4（1=画廊式留白，10=驾驶舱式高密度）

**AI 指令：** 所有生成的标准基线严格为 (8, 6, 4)。不要让用户编辑此文件。但要始终听从用户：根据用户在对话中明确提出的需求动态调整这些值。将这些基线（或用户覆盖的值）作为全局变量，驱动第 3 至第 7 节中的具体逻辑。

## 2. 默认架构与约定
除非用户明确指定不同的技术栈，否则遵循以下结构约束以保持一致性：

* **依赖验证 [强制]：** 导入任何第三方库（如 `framer-motion`、`lucide-react`、`zustand`）之前，必须检查 `package.json`。如果包不存在，必须先输出安装命令（如 `npm install package-name`）再提供代码。**绝不**假设库已安装。
* **框架与交互：** React 或 Next.js。默认使用服务端组件（`RSC`）。
    * **RSC 安全规则：** 全局状态仅在客户端组件中生效。在 Next.js 中，用 `"use client"` 组件包裹 Provider。
    * **交互隔离：** 如果第 4 节或第 7 节（动效/液态玻璃）被激活，相关的交互式 UI 组件必须被提取为独立的叶子组件，并在最顶部添加 `'use client'`。服务端组件只负责渲染静态布局。
* **状态管理：** 隔离的 UI 使用 `useState`/`useReducer`。全局状态仅用于避免深层 prop 透传。
* **样式策略：** 90% 的样式使用 Tailwind CSS（v3/v4）。
    * **Tailwind 版本锁定：** 先检查 `package.json`。不要在 v3 项目中使用 v4 语法。
    * **T4 配置守卫：** 对于 v4，不要在 `postcss.config.js` 中使用 `tailwindcss` 插件。使用 `@tailwindcss/postcss` 或 Vite 插件。
* **禁用 Emoji [关键]：** 绝对不要在代码、标记、文本内容或 alt 文本中使用 emoji。用高品质图标（Radix、Phosphor）或简洁的 SVG 原语替代。Emoji 被禁止。
* **响应式与间距：**
  * 统一断点（`sm`、`md`、`lg`、`xl`）。
  * 页面布局使用 `max-w-[1400px] mx-auto` 或 `max-w-7xl` 约束。
  * **视口稳定性 [关键]：** 绝不使用 `h-screen` 做全高 Hero 区域。始终使用 `min-h-[100dvh]` 防止移动端浏览器（iOS Safari）的灾难性布局跳动。
  * **Grid 优于 Flex 计算：** 绝不使用复杂的 flexbox 百分比计算（`w-[calc(33%-1rem)]`）。始终使用 CSS Grid（`grid grid-cols-1 md:grid-cols-3 gap-6`）构建可靠结构。
* **图标：** 必须使用 `@phosphor-icons/react` 或 `@radix-ui/react-icons` 作为导入路径（先检查已安装版本）。全局统一 `strokeWidth`（如统一使用 `1.5` 或 `2.0`）。


## 3. 设计工程指令（偏差纠正）
大语言模型对特定 UI 陈词套路有统计偏差。使用以下工程化规则主动构建高端界面：

**规则 1：确定性排版**
* **标题/大字：** 默认使用 `text-4xl md:text-6xl tracking-tighter leading-none`。
    * **反平庸：** 对于"高端"或"创意"风格，不鼓励使用 `Inter` 字体。强制使用 `Geist`、`Outfit`、`Cabinet Grotesk` 或 `Satoshi` 等具有独特性格的字体。
    * **技术 UI 规则：** 仪表盘/软件 UI 严禁使用衬线字体。这些场景专用高端无衬线配对（`Geist` + `Geist Mono` 或 `Satoshi` + `JetBrains Mono`）。
* **正文/段落：** 默认使用 `text-base text-gray-600 leading-relaxed max-w-[65ch]`。

**规则 2：色彩校准**
* **约束：** 最多 1 个强调色。饱和度 < 80%。
* **紫色禁令：** "AI 紫/蓝"美学被严格禁止。不要有紫色按钮光晕、不要有霓虹渐变。使用绝对中性底色（Zinc/Slate）搭配高对比度的单一强调色（如翡翠绿、电光蓝或深玫红）。
* **色彩一致性：** 整个输出坚持一套色板。不要在同一项目中在暖灰和冷灰之间摇摆。

**规则 3：布局多样化**
* **反居中偏差：** 当 `DESIGN_VARIANCE > 4` 时，居中的 Hero/H1 区域被严格禁止。强制使用"分屏"（50/50）、"左对齐内容/右对齐素材"或"非对称留白"结构。

**规则 4：质感、阴影与"反卡片滥用"**
* **仪表盘硬化：** 当 `VISUAL_DENSITY > 7` 时，通用卡片容器被严格禁止。使用 `border-t`、`divide-y` 或纯粹的负空间进行逻辑分组。数据指标应当有呼吸感，不要被框住，除非高度（z-index）在功能上确实需要。
* **执行：** 仅在高度传达层级关系时使用卡片。使用阴影时，将其色调调至背景色。

**规则 5：交互 UI 状态**
* **强制生成：** 大语言模型天然倾向生成"静态"的成功状态。你必须实现完整的交互周期：
  * **加载态：** 骨架屏加载器，匹配布局尺寸（避免通用圆形加载动画）。
  * **空状态：** 精心设计的空状态，指引用户如何填充数据。
  * **错误状态：** 清晰的行内错误提示（如表单）。
  * **触感反馈：** `:active` 时使用 `-translate-y-[1px]` 或 `scale-[0.98]` 模拟物理按压感。

**规则 6：数据与表单模式**
* **表单：** 标签必须置于输入框上方。辅助文本可选但应存在于标记中。错误文本在输入框下方。输入块使用标准 `gap-2`。

## 4. 创意主动性（反平庸实现）
为积极对抗通用 AI 设计，系统性地将以下高端编码概念作为基线实现：
* **"液态玻璃"折射：** 需要毛玻璃效果时，超越 `backdrop-blur`。添加 1px 内边框（`border-white/10`）和微妙的内阴影（`shadow-[inset_0_1px_0_rgba(255,255,255,0.1)]`）模拟物理边缘折射。
* **磁性微物理（MOTION_INTENSITY > 5 时）：** 实现按钮向鼠标光标方向微微吸引。**关键：** 绝不使用 React `useState` 做磁性悬停或持续动画。专门使用 Framer Motion 的 `useMotionValue` 和 `useTransform`，在 React 渲染周期之外运行，防止移动端性能崩溃。
* **永续微交互：** 当 `MOTION_INTENSITY > 5` 时，在标准组件（头像、状态点、背景）中嵌入持续的无限循环微动画（脉冲、打字机、漂浮、微光、轮播）。对所有交互元素应用高级弹簧物理（`type: "spring", stiffness: 100, damping: 20`）——禁止线性缓动。
* **布局过渡：** 始终使用 Framer Motion 的 `layout` 和 `layoutId` 属性实现平滑的重排序、尺寸变化和跨状态的共享元素过渡。
* **交错编排：** 不要一次性挂载列表或网格。使用 `staggerChildren`（Framer）或 CSS 级联（`animation-delay: calc(var(--index) * 100ms)`）创建序列化的瀑布式揭示。**关键：** 对于 `staggerChildren`，父级（`variants`）和子级必须位于同一个客户端组件树中。如果数据是异步获取的，将数据作为 props 传入集中的父级 Motion 包装器。

## 5. 性能护栏
* **DOM 开销：** 颗粒/噪点滤镜只应用在固定定位、pointer-events-none 的伪元素上（如 `fixed inset-0 z-50 pointer-events-none`），绝不应用在滚动容器上，以防止持续 GPU 重绘和移动端性能退化。
* **硬件加速：** 绝不动画化 `top`、`left`、`width` 或 `height`。只通过 `transform` 和 `opacity` 做动画。
* **Z-Index 克制：** 绝不随意使用 `z-50` 或 `z-10`。z-index 仅用于系统性的图层上下文（粘性导航栏、模态框、覆盖层）。

## 6. 技术参考（刻度盘定义）

### DESIGN_VARIANCE（设计变化度 1-10）
* **1-3（可预测）：** Flexbox `justify-center`，严格的 12 列对称网格，等距内边距。
* **4-7（偏移）：** 使用 `margin-top: -2rem` 重叠，不同的图片宽高比（如 4:3 与 16:9 相邻），左对齐标题配居中对齐数据。
* **8-10（非对称）：** 瀑布流布局，CSS Grid 使用分数单位（如 `grid-template-columns: 2fr 1fr 1fr`），大面积留白区域（`padding-left: 20vw`）。
* **移动端覆盖：** 对于 4-10 级，任何 `md:` 以上的非对称布局必须在 `< 768px` 视口上激进回退为严格的单列布局（`w-full`、`px-4`、`py-8`），防止水平滚动和布局破碎。

### MOTION_INTENSITY（动效强度 1-10）
* **1-3（静态）：** 无自动动画。仅 CSS `:hover` 和 `:active` 状态。
* **4-7（流畅 CSS）：** 使用 `transition: all 0.3s cubic-bezier(0.16, 1, 0.3, 1)`。加载时使用 `animation-delay` 级联。严格限制在 `transform` 和 `opacity`。谨慎使用 `will-change: transform`。
* **8-10（高级编排）：** 复杂的滚动触发动画或视差效果。使用 Framer Motion hooks。绝不使用 `window.addEventListener('scroll')`。

### VISUAL_DENSITY（视觉密度 1-10）
* **1-3（画廊模式）：** 大量留白。巨大的区块间距。一切感觉非常昂贵和干净。
* **4-7（日常应用模式）：** 标准 Web 应用的常规间距。
* **8-10（驾驶舱模式）：** 极小内边距。不用卡片框；只用 1px 线条分隔数据。所有内容紧密排列。**强制：** 所有数字使用等宽字体（`font-mono`）。

## 7. AI 痕迹（禁止模式）
为保证输出高端、非模板化，你必须严格避免以下常见 AI 设计特征，除非用户明确要求：

### 视觉与 CSS
* **禁止霓虹/外发光：** 不要使用默认的 `box-shadow` 发光。使用内边框或微妙的着色阴影。
* **禁止纯黑：** 绝不使用 `#000000`。使用近黑色、Zinc-950 或炭灰色。
* **禁止过饱和强调色：** 降低强调色饱和度，使其与中性色优雅融合。
* **禁止过度渐变文字：** 不要对大标题使用 text-fill 渐变。
* **禁止自定义鼠标光标：** 过时且损害性能/无障碍性。

### 排版
* **禁止 Inter 字体：** 已被禁止。使用 `Geist`、`Outfit`、`Cabinet Grotesk` 或 `Satoshi`。
* **禁止过大 H1：** 第一个标题不应过于夸张。用字重和颜色控制层级，而非仅靠巨大尺寸。
* **衬线体约束：** 衬线字体仅用于创意/编辑类设计。绝不在干净的仪表盘上使用衬线字体。

### 布局与间距
* **完美对齐与间距：** 确保 padding 和 margin 数学精确。避免元素浮动产生的尴尬间隙。
* **禁止三列卡片布局：** 通用的"三个等宽水平卡片"功能展示行被禁止。使用双列 Z 字型、非对称网格或水平滚动方式替代。

### 内容与数据（"张三"效应）
* **禁止通用姓名：** "张三"、"李四"、"王小明"被禁止。使用高度创意、听起来真实的名字。
* **禁止通用头像：** 不要使用标准 SVG "蛋形"或 Lucide 用户图标做头像。使用创意的、可信的照片占位符或特定样式。
* **禁止假数据：** 避免可预测的输出如 `99.99%`、`50%` 或简单电话号码（`1234567`）。使用有机的、不整齐的数据（`47.2%`、`+86 138-2847-1928`）。
* **禁止创业公司套话命名：** "Acme"、"Nexus"、"SmartFlow"。发明高端的、有语境的品牌名。
* **禁止填充词：** 避免 AI 文案陈词如"赋能"、"无缝"、"释放"、"下一代"。使用具体动词。

### 外部资源与组件
* **禁止失效的 Unsplash 链接：** 不要使用 Unsplash。使用可靠的绝对占位符如 `https://picsum.photos/seed/{random_string}/800/600` 或 SVG UI 头像。
* **shadcn/ui 定制：** 可以使用 `shadcn/ui`，但绝不以默认通用状态使用。必须自定义圆角、颜色和阴影以匹配高端项目美学。
* **生产级整洁：** 代码必须极度干净、视觉惊艳、令人难忘，且每个细节都经过精心打磨。

## 8. 创意武器库（高端灵感）
不要默认生成通用 UI。从以下高级概念库中汲取灵感，确保输出视觉惊艳且令人难忘。适当时使用 **GSAP（ScrollTrigger/视差）** 做复杂滚动叙事，或 **ThreeJS/WebGL** 做 3D/Canvas 动画，而非基础 CSS 动效。**关键：** 绝不在同一组件树中混用 GSAP/ThreeJS 和 Framer Motion。UI/Bento 交互默认使用 Framer Motion。GSAP/ThreeJS 专门用于隔离的全页滚动叙事或 Canvas 背景，包裹在严格的 useEffect 清理块中。

### 标准 Hero 范式
* 停止做暗色图片上居中文字的老套路。尝试非对称 Hero 区域：文字干净地左对齐或右对齐。背景使用高品质的相关图片，配合微妙的风格化渐隐（根据亮/暗模式，优雅地渐入背景色）。

### 导航与菜单
* **Mac OS Dock 放大效果：** 边缘导航栏；图标悬停时流畅缩放。
* **磁性按钮：** 按钮物理性地向光标方向吸引。
* **粘液菜单：** 子项从主按钮像粘性液体般剥离。
* **灵动岛：** 药丸形 UI 组件，变形以显示状态/警告。
* **上下文径向菜单：** 从精确点击坐标展开的圆形菜单。
* **浮动快速拨号：** FAB 弹出成弧形的二级操作按钮行。
* **超级菜单揭示：** 全屏下拉，交错淡入复杂内容。

### 布局与网格
* **Bento 网格：** 非对称、瓷砖式分组（如 Apple 控制中心）。
* **瀑布流布局：** 无固定行高的交错网格（如 Pinterest）。
* **色度网格：** 网格边框或瓷砖显示微妙的、持续动画的颜色渐变。
* **分屏滚动：** 两个屏幕半区在滚动时向相反方向滑动。
* **幕帘揭示：** Hero 区域在滚动时像幕帘一样从中间分开。

### 卡片与容器
* **视差倾斜卡片：** 追踪鼠标坐标的 3D 倾斜卡片。
* **聚光边框卡片：** 卡片边框在光标下方动态发光。
* **毛玻璃面板：** 真实的磨砂玻璃效果，带内折射边框。
* **全息箔卡片：** 虹彩的、彩虹光反射在悬停时移动。
* **Tinder 滑动卡堆：** 用户可以滑走的物理卡片堆。
* **变形模态框：** 按钮无缝扩展成自己的全屏对话容器。

### 滚动动画
* **粘性滚动堆叠：** 卡片粘在顶部并物理性地堆叠覆盖。
* **水平滚动劫持：** 垂直滚动转换为平滑的水平画廊平移。
* **机车滚动序列：** 视频/3D 序列的帧率直接绑定到滚动条。
* **缩放视差：** 中央背景图片在滚动时无缝放大/缩小。
* **滚动进度路径：** SVG 矢量线/路线随用户滚动自动绘制。
* **液态滑动过渡：** 页面过渡像粘性液体一样擦除屏幕。

### 画廊与媒体
* **穹顶画廊：** 像全景穹顶一样感觉的 3D 画廊。
* **封面流轮播：** 中心聚焦、边缘后倾的 3D 轮播。
* **拖拽平移网格：** 可自由向任意方向拖拽的无边界网格。
* **手风琴图片滑块：** 窄的垂直/水平图片条带，悬停时完全展开。
* **悬停图片轨迹：** 鼠标身后留下一连串弹出/淡出的图片。
* **故障效果图片：** 悬停时短暂的 RGB 通道偏移数码失真。

### 排版与文字
* **动态跑马灯：** 无限文字带，滚动时反转方向或加速。
* **文字遮罩揭示：** 大型排版作为视频背景的透明窗口。
* **文字乱码效果：** 加载或悬停时矩阵式字符解码。
* **圆形文字路径：** 文字沿旋转的圆形路径弯曲。
* **渐变描边动画：** 轮廓文字带持续沿描边运动的渐变。
* **动力排版网格：** 字母在光标靠近时闪避或旋转的网格。

### 微交互与特效
* **粒子爆炸按钮：** CTA 按钮成功时碎裂成粒子。
* **液态下拉刷新：** 移动端重载指示器表现为分离的水滴。
* **骨架微光：** 移动的光反射滑过占位框。
* **方向感知悬停按钮：** 悬停填充从鼠标进入的精确方向进入。
* **涟漪点击效果：** 从精确点击坐标扩散的视觉波纹。
* **SVG 线条绘制动画：** 矢量实时绘制自己的轮廓。
* **网格渐变背景：** 有机的、熔岩灯般的动画色块。
* **镜头模糊景深：** 动态聚焦模糊背景 UI 图层以突出前景操作。

## 9. "动效引擎" Bento 范式
生成现代 SaaS 仪表盘或功能展示区域时，必须采用以下 "Bento 2.0" 架构和动效哲学。这超越了静态卡片，强制执行一种"Vercel 内核 + Dribbble 整洁"美学，大量依赖永续物理效果。

### A. 核心设计哲学
* **美学：** 高端、极简、功能性。
* **色板：** 背景使用 `#f9fafb`。卡片为纯白（`#ffffff`），1px 边框色 `border-slate-200/50`。
* **表面：** 所有主要容器使用 `rounded-[2.5rem]`。应用"扩散阴影"（非常轻的、大面积扩散阴影，如 `shadow-[0_20px_40px_-15px_rgba(0,0,0,0.05)]`）创建层次感但不杂乱。
* **排版：** 严格使用 `Geist`、`Satoshi` 或 `Cabinet Grotesk` 字体栈。标题使用微妙的字距（`tracking-tight`）。
* **标签：** 标题和描述必须放在卡片**外部下方**，保持干净的画廊式呈现。
* **像素完美：** 卡片内部使用宽裕的 `p-8` 或 `p-10` 内边距。

### B. 动画引擎规格（永续动效）
所有卡片必须包含**"永续微交互"**。使用以下 Framer Motion 原则：
* **弹簧物理：** 禁止线性缓动。使用 `type: "spring", stiffness: 100, damping: 20` 获得高端、有重量感的手感。
* **布局过渡：** 大量使用 `layout` 和 `layoutId` 属性确保平滑的重排序、尺寸变化和跨状态共享元素过渡。
* **无限循环：** 每张卡片必须有一个无限循环的"活跃状态"（脉冲、打字机、漂浮或轮播），确保仪表盘感觉"活的"。
* **性能：** 用 `<AnimatePresence>` 包裹动态列表并优化到 60fps。**性能关键：** 任何永续动效或无限循环必须被 memoize（React.memo）并完全隔离在自己的微型客户端组件中。绝不触发父级布局的重新渲染。

### C. 5 种卡片原型（微动画规格）
构建 Bento 网格时实现这些特定微动画（如第 1 行：3 列 | 第 2 行：2 列 70/30 分割）：
1. **智能列表：** 带无限自动排序循环的垂直堆叠项。项使用 `layoutId` 交换位置，模拟 AI 实时优先级排序。
2. **命令输入：** 带多步打字机效果的搜索/AI 栏。循环复杂提示词，包含闪烁光标和带微光加载渐变的"处理中"状态。
3. **实时状态：** 带"呼吸"状态指示器的调度界面。包含以"过冲"弹簧效果弹出的通知徽章，停留 3 秒后消失。
4. **宽数据流：** 数据卡片或指标的水平"无限轮播"。确保循环无缝（使用 `x: ["0%", "-100%"]`），速度感觉毫不费力。
5. **上下文 UI（聚焦模式）：** 文档视图，动画化地交错高亮文本块，随后"漂入"带微型图标的浮动操作工具栏。
